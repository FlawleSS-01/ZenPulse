# ZenPulse: AI Meditation App

Прототип мобильного приложения для медитаций с AI-функциями, построенный на React Native + Expo.

## Функционал

### 1. Paywall (Экран подписки)
- Премиальный дизайн с градиентами в стиле медитаций
- Список преимуществ Premium-версии
- Два тарифа: Месячный (599 ₽/мес) / Годовой (2 990 ₽/год) с визуальным выделением выгодного
- Кнопка «Попробовать бесплатно» — имитирует успешную покупку и открывает доступ

### 2. Медитации (Главный экран)
- Сетка карточек медитаций (2 колонки) с эмодзи, названием, описанием и длительностью
- Логика «Заблокировано»: без подписки часть карточек отображается приглушённо (с замком) и при нажатии ведёт на Paywall
- Кнопка Premium в шапке для быстрого перехода на Paywall

### 3. AI Настрой дня
- Три смайлика для выбора настроения: 😊 Радостно / 😌 Спокойно / 😔 Грустно
- При выборе «генерируется» персональная аффирмация с анимацией загрузки
- Возможность перегенерировать текст кнопкой «Сгенерировать другой»

## Запуск

```bash
cd ZenPulse
npm install
npx expo start
```

Далее откройте Expo Go на устройстве и отсканируйте QR-код, либо нажмите `a` для Android-эмулятора / `i` для iOS-симулятора.

## Технологии

- **React Native** + **Expo SDK 55**
- **React Navigation** (Native Stack + Bottom Tabs)
- **expo-linear-gradient** — градиенты
- **@expo/vector-icons (Ionicons)** — иконки
- **React Context API** — управление состоянием подписки

### Ответ на вопрос

ИИ хуже всего справляется со следующими специфическими проблемами мобильной вёрстки:

1. ИИ склонен генерировать фиксированные значения ширины/высоты (например, `width: 375`), которые идеально подходят под один конкретный размер экрана, но ломают верстку на остальных. На iPhone SE (ширина 320pt) контент обрезается, а на Pro Max (ширина 430pt) появляются неоправданные пустые зоны. **Решение в проекте**: вместо абсолютных размеров используется `flex: 1`, процентные значения и `Dimensions.get('window')` для расчёта ширины карточек (`CARD_WIDTH = (width - padding) / 2`), чтобы сетка адаптировалась к любому экрану.

2. ИИ часто забывает про аппаратные вырезы, скругления экрана и домашний индикатор на безрамочных iPhone. Контент «залезает» под статус-бар или перекрывается нижним индикатором. **Решение в проекте**: во всех экранах используется `useSafeAreaInsets()` из `react-native-safe-area-context` — отступы `paddingTop: insets.top` и `paddingBottom: insets.bottom` рассчитываются динамически для каждого устройства.

3. На маленьких экранах длинные заголовки и описания могут выходить за границы карточек или налезать друг на друга. ИИ не тестирует текст на реальных разрешениях и не учитывает длину кириллических строк (русский текст в среднем на 20-30% длиннее английского). **Решение в проекте**: для всех текстовых блоков указаны `numberOfLines` с обрезкой, а карточки используют `minHeight` вместо фиксированной высоты, чтобы растягиваться при необходимости.

### Как ИИ справился с мобильной спецификой

#### Навигация

ИИ корректно реализовал двухуровневую навигацию, характерную для мобильных приложений: `NativeStackNavigator` для модального перехода на Paywall и `BottomTabNavigator` для переключения между основными экранами (Медитации и AI Настрой). При этом были учтены следующие мобильные нюансы:

1. **Абсолютное позиционирование таб-бара**. Нижняя панель навигации использует `position: 'absolute'`, чтобы она «плавала» поверх контента, а не сдвигала его. Это стандартный мобильный паттерн, и ИИ правильно добавил компенсирующий `paddingBottom: insets.bottom + 90` к спискам, чтобы последние карточки не скрывались за панелью.

2. **Платформозависимая высота таб-бара**. На iOS системный домашний индикатор требует дополнительного пространства снизу, поэтому высота таб-бара задана условно: `height: Platform.OS === 'ios' ? 88 : 65` и `paddingBottom: Platform.OS === 'ios' ? 28 : 10`. Это предотвращает наложение кнопок навигации на системный индикатор жестов.

3. **Навигация между стеками**. Paywall использует `navigation.replace('MainTabs')` вместо `navigation.navigate(...)`, чтобы пользователь не мог вернуться на экран оплаты кнопкой «назад» после успешной «покупки» — это важный UX-паттерн для мобильных paywall-экранов.

#### Отступы SafeArea

Работа с безопасными зонами — одна из самых частых ошибок при мобильной вёрстке, и ИИ справился с ней следующим образом:

1. **Глобальный SafeAreaProvider**. В корневом `App.js` всё приложение обёрнуто в `<SafeAreaProvider>`, который определяет размеры безопасной зоны устройства и передаёт их через контекст всем дочерним компонентам.

2. **Поэкранное применение insets**. Каждый экран вызывает `useSafeAreaInsets()` и применяет динамические отступы индивидуально: `paddingTop: insets.top + 10` для отступа от статус-бара/Dynamic Island, `paddingBottom: insets.bottom + 20` (или `+ 90` при наличии таб-бара) для отступа от домашнего индикатора. Число `+ 10` / `+ 20` — дополнительный визуальный «воздух», чтобы контент не прилипал вплотную к системным элементам.

3. **Кнопка закрытия на Paywall**. Крестик для закрытия Paywall позиционируется как `top: insets.top + 10`, а не с фиксированным отступом. Это гарантирует, что кнопка будет доступна и на iPhone SE (без выреза, `insets.top ≈ 20pt`), и на iPhone 15 Pro Max (с Dynamic Island, `insets.top ≈ 59pt`).

4. **ScrollView вместо статичного View**. Все экраны используют `ScrollView` или `FlatList` с `contentContainerStyle`, в котором заданы insets. Это критически важно для маленьких экранов: даже если контент не помещается на iPhone SE, пользователь может прокрутить, а на большом экране всё отображается без скролла.
